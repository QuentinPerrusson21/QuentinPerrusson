TalentHub

Site vitrine full-stack avec back-office admin pour une recherche d'emploi

En ligne Open Source

Contexte & Objectif

TalentHub est un projet de développement web full-stack consistant à créer un site vitrine professionnel avec back-office d'administration pour pouvoir partager des emplois. L'objectif était de développer une application web complète avec gestion de contenu dynamique, un formulaire de contact intelligent avec validation en temps réel, et une interface d'administration complète .

Besoin technique : Créer une application Laravel full-stack avec back-office admin (Filament), formulaire Livewire temps réel, base de données relationnelle (6 tables métier), système d'emails, anti-spam, SEO optimisé et environnement Docker.

Fonctionnalités Développées

  • 🌐 Site vitrine 11 pages

    Accueil avec hero full-screen, offres coup de cœur, compteurs animés, carrousel partenaires. Pages dédiées : Hôtels, Solutions, Talents (processus en 7 étapes), Diamond Private, Partenaires, Équipe, Contact, Offres, Mentions légales, RGPD.

  • ⚙️ Back-office Filament complet

    6 ressources CRUD (offres, équipe, partenaires, messages, pages, paramètres). Dashboard avec widgets de statistiques temps réel et derniers messages. Drag & drop pour l'ordonnancement. Badge de notification des messages non lus.

  • 📨 Formulaire de contact Livewire

    Validation en temps réel champ par champ côté serveur. Protection anti-spam double couche (honeypot + rate limiting 3 envois/heure). Envoi d'email automatique avec template HTML stylisé. Stockage en base de données avec notification admin.

  • 🔍 SEO & Performance

    Meta tags dynamiques par page, Open Graph, Twitter Cards. Commande artisan de génération automatique du sitemap XML. Lazy loading des images, cache des requêtes fréquentes, middleware cache-control.

  • 🧪 Tests automatisés

    5 suites de tests avec Pest/PHPUnit : routes publiques, formulaire de contact, modèles Eloquent, SEO, accès admin Filament. Factories pour chaque modèle.

  • 🐳 DevOps Docker

    Environnement complet avec docker-compose : app PHP, MySQL, Mailpit (capture d'emails), Node.js. Makefile avec 15 commandes automatisées (install, test, deploy, etc.).

Architecture du Projet

  • 📁 Structure MVC Laravel

    7 Modèles Eloquent avec scopes, casts et accesseurs (User, JobOffer, TeamMember, Partner, ContactMessage, Page, SiteSetting).
    7 Contrôleurs pour les pages publiques (Home, Job, Team, Partner, Contact, Page).
    9 Migrations pour la structure de la base de données.
    7 Seeders avec données réalistes.

  • 🎨 Frontend composants Blade

    8 composants réutilisables : navbar sticky avec transparence au scroll, hero full-screen avec overlay, cartes d'offres, cartes équipe, logos partenaires avec hover reveal, compteurs animés, titres de section, footer multi-colonnes.

  • 🗄️ Base de données relationnelle

    6 tables métier : offres d'emploi (avec slug auto-généré, enum contrat, système coup de cœur), messages de contact (avec statut lu/non-lu), pages de contenu statique (mentions légales, RGPD), paramètres clé-valeur du site.

Challenges Techniques Rencontrés

  • Configuration de l'environnement

    Défi : Faire fonctionner Laravel avec Docker (MySQL, Mailpit) en local, gérer les dossiers storage manquants qui faisaient crasher Composer.
    Solution : Ajout de scripts automatiques dans composer.json (post-autoload-dump) qui créent les dossiers storage avant le démarrage de Laravel, et fallback dans config/view.php pour éviter les erreurs realpath().

  • Back-office Filament

    Défi : Créer 6 ressources CRUD complètes avec upload d'images, drag & drop, filtres, badges et widgets dashboard.
    Solution : Utilisation de l'architecture Resource/Pages de Filament 3, avec auto-discovery des ressources et widgets dans le PanelProvider.

  • Formulaire de contact sécurisé

    Défi : Implémenter une validation temps réel sans JavaScript, avec protection anti-spam efficace.
    Solution : Livewire 3 avec wire:model.blur pour la validation champ par champ, honeypot (champ invisible piège à bots) et RateLimiter Laravel (3 envois/heure max par IP).

  • SEO & Performance

    Défi : Optimiser le référencement avec des meta tags dynamiques et un sitemap auto-généré.
    Solution : Variables Blade transmises au layout pour les meta/OG/Twitter Cards, commande Artisan personnalisée avec Spatie Sitemap, lazy loading des images et cache Eloquent.

Rétrospective Personnelle

Ce que j'ai appris :

  • Architecture MVC complète avec Laravel 11 (routes, contrôleurs, modèles Eloquent, vues Blade)
  • Création d'un back-office admin professionnel avec Filament 3 (CRUD, widgets, dashboard)
  • Composants interactifs côté serveur avec Livewire 3 (validation temps réel, sans JavaScript)
  • Design responsive mobile-first avec Tailwind CSS et interactivité légère avec Alpine.js
  • Base de données relationnelle avec migrations, seeders, factories et scopes Eloquent
  • Sécurité web : protection CSRF, honeypot anti-spam, rate limiting, validation côté serveur
  • DevOps : conteneurisation Docker, Makefile, gestion d'environnement avec .env
  • Tests automatisés avec Pest/PHPUnit (routes, formulaires, modèles, SEO, admin)
  • SEO technique : meta tags dynamiques, Open Graph, sitemap XML, lazy loading, cache
