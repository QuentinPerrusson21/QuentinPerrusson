Contexte & Objectif

Le site d'origine reposait sur le Website Builder IA d'Hostinger. Rapide à déployer, cet outil s'est révélé très limitant : code propriétaire et fermé empêchant toute personnalisation CSS/JS avancée, structure HTML sémantiquement pauvre (plusieurs balises <h1> par page) pénalisant le référencement, et une administration qui ne permettait pas au client de gérer son contenu en autonomie.

La refonte : migration intégrale vers WordPress (CMS open-source, garantissant la pérennité et la propriété des données), associé au constructeur Elementor Pro pour un contrôle total du design et une interface d'édition intuitive côté client.

L'objectif final d'un projet mené sur 9 mois : au-delà de la livraison technique, rendre le client totalement autonome sur la gestion de son nouvel environnement.

Front-End & Responsive Design (CSS sur-mesure)

Au-delà des outils de base d'Elementor, l'exigence artistique du client a nécessité l'injection de CSS sur-mesure, particulièrement pour l'adaptation mobile.

  • Maîtrise du Box Model (padding vs. margin)

    Problème : sur mobile, des blocs de texte sur fond blanc touchaient les bords de l'écran, dégradant la lisibilité et l'aspect « premium ».

    Solution : ajustement précis des padding (le texte « respire » dans son conteneur) et des margin (décollage des bords). Des marges négatives ont permis des effets de chevauchement texte/image (parallaxe statique), la superposition étant gérée via z-index.

  • La « Carte Événement » (module agenda)

    Problème : l'affichage par défaut des dates de conférences s'écrasait et devenait illisible sur petits écrans, occupant 100 % de la largeur sans structuration.

    Solution : CSS spécifique via Media Queries transformant le bloc en véritable « carte » centrée et encadrée.

    @media (max-width: 768px) {
    .event-card {
        width: 85%;
        margin: 0 auto;
        border: 2px solid #638eb1;
    }
}

  • Résolution du « syndrome de l'image élastique »

    Problème : les photographies d'art et d'apnée se déformaient (étirement/écrasement) selon la largeur des smartphones (iPhone Mini vs. Samsung Ultra).

    Solution : object-fit: cover couplé à width: 100% et height: auto pour remplir le conteneur sans altérer l'aspect ratio. Pour les galeries carrées, ajout de aspect-ratio: 1 / 1.

  • Conteneurs d'arrière-plan (images de fond)

    Problème : un conteneur avec image d'eau en arrière-plan s'aplatissait sur mobile, l'image n'apparaissant plus que comme une fine ligne.

    Solution : forçage de l'ouverture du conteneur via min-height (ex. min-height: 250px), combiné à background-size: cover et background-position: center center.

  • Contrôle typographique (« Mobile Break »)

    Problème : les titres longs comme « Clément Lesaffre » se coupaient aléatoirement sur mobile, nuisant à l'image de marque.

    Solution : insertion d'un <span class="mobile-break"> dans le texte, activé uniquement sur mobile pour un retour à la ligne propre. Sur desktop, le span reste inline (inactif).

    @media (max-width: 767px) {
    .mobile-break { display: block !important; }
}

Fonctionnalités, Sécurité & Optimisation

  • 🛡️ Sécurité & lutte anti-spam (formulaires)

    Problème : le formulaire de contact était ciblé par des robots, générant un flux important de spam.

    Solution — défense à deux niveaux :

    • Honeypot : champ masqué en CSS ; rempli par les robots qui scannent le HTML, il déclenche le blocage automatique de l'envoi.
    • Google reCAPTCHA v3 : validation invisible analysant le comportement en arrière-plan, sans énigme visuelle pour les utilisateurs légitimes.

  • ⚖️ Conformité légale (RGPD)

    Site professionnel devant être conforme à la réglementation européenne : installation et configuration d'une extension de gestion du consentement des cookies (bandeau), et création d'une page Mentions Légales (hébergeur, traitement des données via le formulaire).

  • ⚡ Optimisation des médias (performance)

    Site riche en médias HD (photos d'art, vidéos d'apnée) — enjeu : temps de chargement rapide sur mobile.

    • Images : conversion systématique en WebP (compression supérieure sans perte visible) et nettoyage des métadonnées EXIF.
    • Vidéos : compression MP4 optimisée et système de Fallback Image en JS : détection des connexions mobiles lentes pour remplacer la vidéo d'arrière-plan par une image fixe légère (gain de bande passante et de LCP).

  • 🔍 Stratégie SEO

    • Restructuration sémantique : une seule balise <h1> par page (règle d'or du référencement).
    • Optimisation On-Page via l'extension Rank Math SEO : renseignement des attributs alt sur toutes les images, travail sur la densité sémantique des mots-clés (apnée, art cyanotype, conférences).
    • Résultat : score d'audit SEO de 92/100, malgré une forte charge visuelle.

Transfert de compétences — Micro-Learning :

L'aboutissement du projet ne résidait pas seulement dans la livraison technique, mais dans le transfert de compétences pour rendre le client autonome.

  • Pédagogie asynchrone : plutôt qu'un long manuel textuel, formation via de très courtes vidéos tutoriels (~30 secondes).
  • Actions ciblées : modifier un tarif, uploader une photo dans une galerie, mettre à jour une date dans l'agenda des conférences.
  • Résultat : le client, auparavant bloqué par l'outil IA, gère désormais son site en toute autonomie.
